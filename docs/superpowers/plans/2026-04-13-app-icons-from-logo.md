# Plan: Generate app icons from logo.svg for iOS, Android, and web

## Context

The mobile app (`apps/mobile/`) currently ships with placeholder/outdated PNG icons in `assets/` (icon.png, adaptive-icon.png, splash-icon.png, favicon.png) that don't match the brand logo at `apps/web/public/logo.svg`. The web app has `app/icon.svg` and a `favicon.ico` but is missing an `apple-icon.png` and a PWA manifest, so iOS home-screen bookmarks fall back to a low-res screenshot.

Goal: regenerate every app icon from the single source `apps/web/public/logo.svg` so web, iOS, and Android all render the brand star consistently. User wants a dark navy (`#111827`) opaque background and a reusable script committed to the repo so icons can be regenerated anytime the logo changes.

## Approach

1. **Add one reusable generation script** at `scripts/generate-icons.sh` that shells out to ImageMagick (already available; used by the station-image skill).
2. **For mobile (Expo SDK 54):** Expo auto-generates all per-platform sizes at build time from a single 1024×1024 master, so we only need to overwrite the four existing PNGs in `apps/mobile/assets/`. No `app.json` changes needed.
3. **For web (Next.js App Router):** use the file-based icon convention — drop `apple-icon.png` and `icon.png` into `apps/web/app/` and add `site.webmanifest` + Android PWA icons in `apps/web/public/`. Next.js automatically serves them with correct `<link rel>` tags; no `layout.tsx` metadata changes required for the file-convention files themselves, but we'll add `manifest: '/site.webmanifest'` to the metadata.

### Background + safe area rules

- All icons: dark navy `#111827` opaque background (matches `apps/web/app/icon.svg`).
- Android adaptive foreground: logo scaled to ~66% of canvas (centered) to stay inside Android's adaptive safe-area circle. Background layer is a separate solid navy PNG.
- iOS: full-bleed, logo at ~80% scale, navy background, no transparency, no rounding (iOS rounds automatically).

## Files to create

### `scripts/generate-icons.sh` (new, executable)

Bash script that:
1. Sets `SRC=apps/web/public/logo.svg`, `BG=#111827`.
2. Renders a 1024×1024 master PNG from the SVG using `magick -background $BG -density 1200 $SRC -resize 1024x1024 -gravity center -extent 1024x1024 /tmp/ctt-icon-master.png`.
3. Renders a padded variant (logo at 66%) for Android adaptive foreground.
4. Writes mobile outputs:
   - `apps/mobile/assets/icon.png` (1024×1024, full-bleed master)
   - `apps/mobile/assets/adaptive-icon.png` (1024×1024, padded foreground on navy)
   - `apps/mobile/assets/splash-icon.png` (1024×1024, padded, matches splash background in `app.json`)
   - `apps/mobile/assets/favicon.png` (48×48)
5. Writes web outputs:
   - `apps/web/app/icon.png` (512×512 — Next.js convention, serves as generic `icon`)
   - `apps/web/app/apple-icon.png` (180×180 — Next.js convention, auto-emits `apple-touch-icon`)
   - `apps/web/public/favicon.ico` (multi-size: 16, 32, 48 — **replaces** current `apps/web/app/favicon.ico`; move the favicon from `app/` to `public/` for clarity, OR keep it in `app/` since Next.js file convention lives there; we'll keep it in `app/favicon.ico` to match the current convention)
   - `apps/web/public/icon-192.png` (192×192, for PWA manifest)
   - `apps/web/public/icon-512.png` (512×512, for PWA manifest)
   - `apps/web/public/apple-touch-icon.png` (180×180, redundant safety net for older iOS Safari that ignores Next.js route)

### `apps/web/public/site.webmanifest` (new)

```json
{
  "name": "Chicago Transit Tracker",
  "short_name": "CTT",
  "icons": [
    { "src": "/icon-192.png", "sizes": "192x192", "type": "image/png" },
    { "src": "/icon-512.png", "sizes": "512x512", "type": "image/png" }
  ],
  "theme_color": "#111827",
  "background_color": "#111827",
  "display": "standalone",
  "start_url": "/"
}
```

## Files to modify

### `apps/web/app/layout.tsx`

Add `manifest: '/site.webmanifest'` to the exported `metadata` object so Next.js emits `<link rel="manifest">`. No other changes — the file-based `app/icon.png` and `app/apple-icon.png` are picked up automatically.

### `apps/mobile/assets/*.png` (overwrite)

`icon.png`, `adaptive-icon.png`, `splash-icon.png`, `favicon.png` — regenerated by the script. `app.json` already points at these paths, so no config change.

### `apps/web/__tests__/` (new test)

Add `apps/web/__tests__/app/layout-metadata.test.ts` asserting that the exported `metadata` includes `manifest: '/site.webmanifest'`. Per the project's testing rule, any source-file change gets a test update.

## Files to reference (not modify)

- `apps/web/public/logo.svg` — source of truth
- `apps/web/app/icon.svg` — existing dark variant, confirms navy background choice
- `apps/mobile/app.json` — already correctly points at `./assets/icon.png`, `./assets/adaptive-icon.png`, etc.
- `.claude/skills/station-image/` (or equivalent) — reference for how ImageMagick is invoked elsewhere in the project

## Verification

1. **Run the script:** `bash scripts/generate-icons.sh` — should complete without errors and produce the 10 output files.
2. **Spot check masters:** `magick identify apps/mobile/assets/icon.png` → should report `1024x1024`, 8-bit sRGB, no alpha channel (or opaque alpha). Repeat for `apps/web/app/apple-icon.png` (180×180).
3. **Visual check:** open `apps/mobile/assets/icon.png`, `apps/mobile/assets/adaptive-icon.png`, and `apps/web/app/apple-icon.png` in Preview — logo should be centered on navy, readable, and the adaptive-icon foreground should have visible padding around the star.
4. **Web:** `pnpm -w run dev`, visit `http://localhost:3000`, view source, confirm `<link rel="icon">`, `<link rel="apple-touch-icon">`, and `<link rel="manifest">` tags are present. Bookmark to iOS home screen via BrowserStack/real device to confirm the apple-touch-icon renders.
5. **Mobile:** `cd apps/mobile && npx expo start`, launch iOS simulator and Android emulator. Home screen should show the new icon on both platforms. Splash screen should show the padded logo on navy.
6. **Tests + lint:** `pnpm -w run test` and `pnpm -w run lint` both clean (standing rule).

## Out of scope

- Redesigning the logo or producing a separate monochrome/tintable variant for iOS 18 tinted mode (can add later as `apps/web/app/icon-tinted.png` if desired).
- Updating `og-default.jpg` — that's the Open Graph share image, a separate asset.
- iOS App Store 1024×1024 marketing icon — Expo generates this from the same `icon.png` at EAS build time.
