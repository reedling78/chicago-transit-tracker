# 404 Page Design Spec

## Context

The site currently has no custom 404 page — Next.js serves a generic one. A custom 404 improves UX by keeping users on-site with helpful navigation, improves SEO with proper metadata, and adds personality through transit-themed humor.

## Design Decision

**Layout:** Hero + Route Cards
**Copy style:** Transit-themed humor — "End of the line" theme

## Page Structure

### File Location

`app/not-found.tsx` — the Next.js App Router convention for custom 404 pages. This is a **server component** (no `'use client'` needed).

### Hero Section (centered)

- **404** — displayed large and bold in CTA Red (`#c60c30`)
- **Headline:** "You've reached the end of the line"
- **Subtext:** "All passengers must exit. The page you're looking for doesn't exist or has been moved."

### Divider

A simple `border-t` separator matching existing site patterns (`border-gray-200 dark:border-gray-800`).

### Popular Destinations Section

Section label: "Popular Destinations" — small caps, uppercase, tracking-widest (matches existing section label pattern from CTA page).

Four `LinkCard` components with colored left-border accents:

| Destination    | href          | subtitle                   | accentColor            |
| -------------- | ------------- | -------------------------- | ---------------------- |
| Home           | `/`           | Back to the main station   | `#c60c30` (CTA Red)    |
| CTA Lines      | `/cta`        | 8 rapid transit rail lines | `#00a1de` (CTA Blue)   |
| Metra Lines    | `/metra`      | 11 commuter rail lines     | `#009b3a` (CTA Green)  |
| Service Alerts | `/cta/alerts` | Live CTA & Metra alerts    | `#f9461c` (CTA Orange) |

The `LinkCard` component already supports `accentColor`, `subtitle`, and arrow indicators — no modifications needed.

## SEO & Metadata

Static `metadata` export (not `generateMetadata` since there are no dynamic params):

```typescript
export const metadata: Metadata = {
  title: 'Page Not Found',
  description:
    'The page you are looking for does not exist. Browse CTA and Metra transit lines, stations, and service alerts.',
  openGraph: {
    title: 'Page Not Found',
    description:
      'The page you are looking for does not exist. Browse CTA and Metra transit lines, stations, and service alerts.',
    url: `${siteConfig.url}/404`,
    images: [{ url: siteConfig.ogImage }],
    type: 'website',
  },
  twitter: {
    card: 'summary_large_image',
    title: 'Page Not Found',
    description:
      'The page you are looking for does not exist. Browse CTA and Metra transit lines, stations, and service alerts.',
    images: [siteConfig.ogImage],
  },
  robots: {
    index: false, // Don't index 404 pages
  },
}
```

Key SEO decisions:

- `robots: { index: false }` — 404 pages should not be indexed by search engines
- Descriptive `description` that mentions site content for any crawlers that do visit
- Full OG/Twitter metadata in case the URL is shared

## Dark Mode

Follows existing patterns:

- 404 number: CTA Red works on both light and dark backgrounds
- Text: `text-gray-900 dark:text-white` for headings, `text-gray-500 dark:text-gray-400` for body
- Section label: `text-gray-400 dark:text-gray-500`
- Divider: `border-gray-200 dark:border-gray-800`
- LinkCard already handles dark mode internally

## Styling Pattern

Matches existing page patterns:

- Content centered with `text-center` for the hero section
- Max width constrained with `max-w-lg mx-auto` for the subtext
- Section uses standard page spacing
- No custom CSS needed — all Tailwind utility classes

## Components Reused

- `LinkCard` (`app/components/LinkCard.tsx`) — navigation cards with accent colors, already supports all needed props
- `siteConfig` (`app/lib/siteConfig.ts`) — site name, URL, OG image

## Components NOT Used

- `PageHeader` — the 404 page has a unique visual treatment (large number + custom copy) that doesn't fit the standard PageHeader pattern
- `Breadcrumb` — doesn't make sense on a 404 since the user's path is invalid

## What This Does NOT Include

- No animations or client-side interactivity (keeps it a server component)
- No sitemap entry (404 pages should not be in the sitemap)
- No `generateStaticParams` (not a dynamic route)

## Testing

- Unit test verifying the component renders the 404 heading, subtext, and all 4 navigation links
- Verify metadata export has all required fields
- Manual verification: visit a non-existent URL in dev mode to confirm the page renders correctly in both light and dark mode
